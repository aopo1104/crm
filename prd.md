一、背景
目前crm有以下问题：
- 业务员角度：
  - 新客
    - 辛苦找到客户的联系方式，很多是被建联了。
      - 很多建联了但是没联系的客户，为了避免被其他业务员联系，也会申请延期。导致很多客户会沉寂，错失机会。
    - 系统没有办法导出报价单，导致出去的报价单没有规范，价格也无法管控。同时效率也不高
暂时无法在乐歌飞书文档外展示此内容
- 管理层角度：
  - 推广层面
    - 下发推广任务只能靠飞书，无法通过系统且后续无法监控
[图片]
    - 每封推广信都需要每个业务员自己拟，造成资源浪费。
[图片]
    - 系统只有结果指标，不知道实际开发新客/推广新产品的情况
  - 进度跟进层面
    - 无法知道客户的实际进度/推广过哪些产品，也不知道什么产品推广的效果比较好。只能靠看一封封邮件&问业务员
    - 所有客户都是扁平的，没办法分层管理，容易造成资源浪费和遗漏。
    - 没有地方能看到客户的全部生命周期。
  - 产品层面
    - 由于只有最终的成交数据，不知道客户对于产品的喜好度，价格接受程度是怎么样的

二、目标：
1. 看清客户
  1. 资料完整：所有与客户相关的关键信息能统一看到；
  2. 过程可见：客户从线索到成交的全过程有记录可追溯；
  3. 价值分层：知道哪些客户更值钱、更值得投入。
2. 看懂销售
  1. 行为记录可视化
  2. 销售漏斗
3. 智能化
  1. 新客：定期联系
  2. 老客：防流失

三、功能点
3.1、客户全生命周期管理（对应目标：看清客户）
1. 客户 360° 统一视图
  - 客户基础信息、联系方式、跟进记录（包括邮件、whatsapp等沟通方式）、邮件、报价、订单、推广记录集中展示
  - 关键资料字段强制完善，确保客户信息完整可查
2. 客户生命周期全链路追溯
  - 线索→建联→跟进→报价→成交→复购→售后全阶段节点记录
  - 每一步操作人、时间、内容自动留痕，可回溯
3. 客户价值分层管理
  - 支持按成交金额、需求强度、跟进潜力、复购频率自定义客户分层（A/B/C 级等）
  - 根据客户购买的产品品类，进行对客户类型的分类。
  - 分层标签可视化，支持按分层筛选、分配、跟进。
4. 客户资源防冲突与去重
  - 打通业务系统/钉富系统的关系
  - 线索 / 客户自动去重，显示「已建联 / 已跟进」状态
  - 禁止重复认领、重复联系
3.2、线索公海与新客盘活（解决业务员：重复建联、客户沉寂痛点）
1. 公海线索规范化管理
  - 公海线索统一池，认领 / 释放规则可配置
  - 实时显示客户建联状态，避免业务员白找联系方式
2. 客户延期与占用管控
  - 规范客户保护 / 延期申请条件、时长
  - 超期未跟进自动回收公海，防止长期占用导致沉寂
3. 沉睡客户智能提醒
  - 长期无跟进客户自动标记 + 提醒
  - 支持批量激活、重新分配，减少商机流失
3.3、报价与价格管控（解决业务员：报价不规范、无导出、效率低）
1. 标准化报价单体系
  - 系统内置报价单模板，支持自定义字段
  - 一键生成、导出（Excel）、打印、发送
2. 价格权限管控
  - 设定产品基准价、折扣范围、最低售价
  - 超权限报价自动触发审批，实现价格可控
3. 报价全流程留痕
  - 所有报价自动关联客户，可查、可对比、可追溯
4. 快速报价
  - 一键关联客户 + 产品，自动填充信息，提升报价效率
3.4、市场推广自动化（解决管理层：推广无监控、重复写推广信）
1. 推广任务系统下发
  - 管理层在线创建推广任务，绑定产品 / 客户分层 / 业务员
  - 系统自动推送
2. 推广任务执行监控
  - 实时查看任务发送率、打开率、客户反馈、跟进情况
  - 未执行 / 逾期任务自动提醒
3. 推广邮件模板库
  - 统一产品推广信、开发信模板库
  - 业务员一键调用 + 微调，避免重复拟写
4. 产品推广追踪
  - 记录每个客户被推广过的产品、次数、反馈
  - 自动统计各产品推广覆盖量
3.5、销售过程管理（对应目标：看懂销售）
1. 销售行为可视化
  - 邮件发送、电话、拜访、推广、报价等行为自动记录
  - 业务员工作行为一目了然，不用逐封查邮件
2. 销售漏斗实时展示
  - 线索量→建联率→跟进率→报价率→成交率全漏斗数据
  - 按人员 / 部门 / 时间维度筛选
3. 客户进度实时看板
  - 客户当前阶段、对接产品、未跟进原因统一查看
  - 不用反复询问业务员，不用翻邮件
4. 过程指标而非仅结果
  - 新增线索、建联数、推广数、报价数等过程数据自动统计
  - 搭配最终成交数据，形成完整评估体系
3.6、数据洞察与分析（解决管理层：不知客户偏好、产品效果）
1. 产品效果分析
  - 各产品推广次数、咨询量、报价次数、成交率统计
  - 自动识别高效果 / 低效果产品
2. 客户偏好与价格接受度分析
  - 基于客户咨询、报价、议价、成交记录分析产品喜好
  - 分析客户对价格的敏感程度与接受区间
3. 多维度管理报表
  - 客户分层报表、销售过程报表、推广效果报表、产品分析报表
  - 支持导出、自定义查看
3.7、智能化运营（对应目标：智能化）
1. 新客智能定期跟进
  - 按客户阶段配置跟进频次，自动推送跟进提醒
  - 防止新客长期不联系被浪费
2. 老客防流失预警
  - 根据互动频率、订单间隔、反馈情况识别流失风险客户
  - 自动预警并推荐跟进策略
3. 智能分配与盘活
  - 沉睡 / 高价值客户智能推荐优先跟进

一、修改线索流程
线索的流程修改为：
待接触 → 接触阶段 → 互联阶段 ->报价阶段 → 样品阶段 → 已成交
核心行为：
1. 待接触：业务员添加线索，未上传跟进记录
2. 接触阶段：业务员上传跟进记录，未上传客户回复记录
3. 互联阶段：业务员上传客户回复记录
4. 报价阶段：业务员从系统导出报价
5. 样品阶段：该客户下存在样品单
6. 已成交：该客户下存在正式订单

业务员可搜索并关联待接触/接触阶段的线索。

二、添加业务员报表
添加业务员关于新客的漏斗报表。


## V1.2 修改PRD

### 一、线索模块改动

#### 1. 阶段调整
- 线索阶段缩减为 4 阶段：**待接触 → 接触阶段 → 互联阶段 → 报价阶段**
- 移除"样品阶段"和"已成交"阶段（线索转为客户后，由客户阶段承接后续流程）

#### 2. 线索分类与可见性
- **公海线索**：无跟进业务员的线索，对所有业务员可见
- **私有线索**：有跟进业务员的线索，仅对跟进业务员可见
- 待接触/接触阶段的线索不设认领人，仅有跟进业务员（支持一对多）

#### 3. 线索查重
- 新增线索查重页面，业务员可查询系统中已存在的线索/客户
- 待接触/接触阶段的线索：可点击「跟进」，该业务员自动加入线索的跟进人列表
- 其他阶段的线索/客户：不可点击跟进

#### 4. 线索操作
- 支持转客户
- 支持添加商机

#### 5. 跟进记录录入
录入字段：
- 跟进时间
- 跟进主题
- 跟进类型（线下拜访 / 会议 / 邮件联系 / 社媒联系 / WhatsApp联系）
- 联系人
- 联系内容

#### 6. 客户回复记录录入
录入字段：
- 回复时间
- 回复内容

---

### 二、客户模块改动

#### 1. 客户阶段
客户阶段为 3 阶段：**未成交 → 样品阶段 → 已成交**

核心行为驱动：
| 阶段 | 触发条件 |
|------|----------|
| 未成交 | 客户未下单 |
| 样品阶段 | 客户下单了样品 |
| 已成交 | 客户下单了正式订单 |

#### 2. 客户详情页增强
- 体现订单记录、商机记录、产品记录、品类记录
- 根据品类对客户进行分组（一个客户可属于多个品类），便于后续推广计划制定
- 显示客户品类偏好，为精准推广提供依据

#### 3. 客户列表操作
- 支持下订单
- 支持下商机单

---

### 三、商机管理

#### 1. 商机创建
- 线索/客户均可添加商机
- 商机可生成报价单

#### 2. 商机字段
- 商机名称
- 关联线索/客户
- 预估金额
- 预计成交时间
- 商机阶段（初步接触 / 需求确认 / 方案报价 / 商务谈判 / 赢单/输单）
- 关联产品/品类
- 备注说明

#### 3. 商机转化
- 商机可生成报价单
- 报价单审批通过后可转订单

### 六、推广任务优化

#### 1. 推广计划创建

#### 2. 推广任务执行
**方式一：手动批量录入**
- 可批量填入推广客户代码
- 手动标记推广状态

**方式二：关联钉富 EDM 系统**
- 关联钉富（邮箱系统）的 EDM 标题

  - 成功数量
  - 打开数量
  - 失败数量
  - 拦截数量
  - 发送状态
- 系统自动抓取 EDM 中的客户代码，录入客户的跟进进度

#### 3. 推广效果跟进
- 推广任务执行后，业务员可按推广计划维度补充客户回复情况
- 以客户维度录入回复内容，自动归档到客户的跟进进度中
